# Gopenclaw 对齐官方 OpenClaw v2026.2.15 改进清单

> 基于官方 [OpenClaw v2026.2.15](https://github.com/openclaw/openclaw/releases/tag/v2026.2.15) 更新日志

## Changes

### 1. Discord Components v2
- [ ] 实现 Components v2 UI (buttons, selects, modals, attachment-backed file blocks)
- [ ] 实现 embeds passthrough
- [ ] 实现 exec approval UX refinements (CV2 containers, button layout, Discord-forwarding skip)

### 2. Plugins Hooks
- [x] 暴露 `llm_input` hook payload - 让扩展可以观察 prompt/input context (HooksConfig.LLMInputHook)
- [x] 暴露 `llm_output` hook payload - 让扩展可以观察 model output usage details (HooksConfig.LLMOutputHook)

### 3. Subagents 嵌套支持
- [x] 实现 nested sub-agents (sub-sub-agents) (通过 Subagent 配置)
- [x] 添加 configurable depth 配置 (`agents.defaults.subagents.maxSpawnDepth: 2`) (SubagentConfig.MaxSpawnDepth)
- [x] 实现 `maxChildrenPerAgent` limit (default 5) (SubagentConfig.MaxChildrenPerAgent)
- [x] 实现 depth-aware tool policy
- [x] 实现 announce chain routing

### 4. Channel Ack Reaction Overrides
- [x] 实现 per-channel ack reaction overrides (account/channel-level) (MessagesConfig.AckReaction)
- [x] 支持 platform-specific emoji formats (Slack/Discord/Telegram)

### 5. Cron/Gateway Webhook
- [x] 添加 finished-run webhook delivery toggle (`notify`) (CronConfig.Notify)
- [x] 添加 dedicated webhook auth token support (`cron.webhookToken`) (CronConfig.WebhookToken)

### 6. Channels Deduplication
- [x] 统一 core + extensions 的 probe/token resolution base types

## Fixes

### 安全修复
- [x] Security: 用 SHA-256 替换 deprecated SHA-1 sandbox configuration hashing
- [x] Security/Logging: 编辑 Telegram bot tokens 从 error messages
- [x] Sandbox/Security: block dangerous sandbox Docker config (bind mounts, host networking, unconfined seccomp/apparmor)
- [x] Security/Exec: replace shell-based execSync with execFileSync
- [x] Security/Media: use crypto.randomBytes() for temp file names
- [x] Security/Gateway: set baseline security headers
- [x] Security/Config: parse frontmatter YAML using YAML 1.2 core schema
- [x] Security/Discord: escape backticks in exec-approval embed content

### Sandbox
- [x] Sandbox: preserve array order in config hashing
- [x] Sandbox: 修复 workspace-directory orphaning during SHA-1 -> SHA-256 slug migration

### Gateway

- [x] Gateway/Control UI: preserve requested operator scopes for bypass modes
- [x] Gateway/Agent: reject malformed agent:-prefixed session keys
- [x] Gateway/Chat: harden chat.send inbound message handling
- [x] Gateway/Send: return actionable error when send targets internal-only webchat

### Agents
- [x] Agents/Security: sanitize workspace paths before embedding into prompts
- [x] Agents/Sandbox: clarify system prompt path guidance
- [x] Agents/Context: apply configured model contextWindow overrides
- [x] Agents/Context: derive lookupContextTokens from auth-available model metadata
- [x] Agents/OpenAI: force store=true for direct OpenAI Responses/Codex runs
- [x] Agents: return explicit timeout error when embedded run times out

### Memory

### Telegram
