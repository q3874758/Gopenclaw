# Gopenclaw 对齐官方 OpenClaw v2026.2.24 改进清单

> 基于官方 [OpenClaw v2026.2.24](https://github.com/openclaw/openclaw/releases/tag/v2026.2.24) 更新日志

## Changes

### 1. 依赖更新
- [x] 刷新运行时和工具包（Go 版本无需关注此）

### 2. 安全模型
- [x] 添加 `security.trust_model.multi_user_heuristic` 标志检测共享用户入口 (SecurityConfig.TrustModel.MultiUserHeuristic)
- [x] 为多用户设置添加指导（sandbox.mode="all", workspace-scoped FS）

### 3. Talk/Gateway 配置
- [x] 添加提供者无关的 Talk 配置 (TalkConfig)
- [ ] 暴露 Gateway Talk ElevenLabs 配置元数据 - 可扩展

### 4. Auto-reply/Abort 快捷键
- [ ] 扩展独立停止短语（`stop openclaw`, `stop action`, `stop run`, `stop agent`, `please stop` 等）
- [ ] 接受尾部标点符号
- [ ] 添加多语言停止关键词

## Breaking Changes

### 5. 安全/沙箱
- [ ] 默认阻止 Docker `network: "container:"` 命名空间加入模式

### 6. Heartbeat 投递
- [ ] Heartbeat 现在阻止直接/DM 目标（当目标解析识别为直接聊天时）
- [ ] DM 投递现在需要显式配置

## Fixes

### 7. Slack/DM 路由
- [x] 将 `D*` channel IDs 视为直接消息

### 8. Telegram/媒体获取
- [x] 优先 IPv4 在 IPv6 之前

### 9. Telegram/回复
- [x] 当 markdown 格式化为空 HTML 时，重试纯文本投递

### 10. Discord
- [x] 线程 channel proxy fetch
- [x] 代理感知的 gateway 元数据获取
- [x] 恢复 block-streamed reply delivery
- [x] 恢复运行时 DAVE 依赖

### 11. WhatsApp
- [x] 抑制标记为 reasoning 的出站 payload
- [x] 将 `Reasoning:` 开头的文本 payload 硬丢弃

### 12. 通道 Typing keepalive
- [x] 在长回复期间刷新 channel typing callbacks

### 13. Messaging tool 去重
- [x] 将发起 channel 元数据视为权威

### 14. Cron/Heartbeat 投递
- [x] 停止继承缓存的 session `lastThreadId`

### 15. Heartbeat 队列
- [x] 当 session 已有活动运行时丢弃 heartbeat 运行

### 16. Heartbeat 默认/提示
- [x] 将隐式 heartbeat 投递目标从 `last` 切换到 `none`

### 17. 安全/路由
- [x] 失败关闭跨通道回复
- [x] 将出站目标解析绑定到当前 turn 的 source channel 元数据

### 18. 路由/会话隔离
- [x] 强化 followup 路由
- [x] 保留队列溢出摘要路由元数据
